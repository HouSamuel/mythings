编写一个 manim 扩展文件，遵守以下要求：

## 自定义函数（注入到 Scene 类）
- 定义一个普通函数，其第一个参数为 self（代表 Scene 实例）。
- 在扩展文件的全局作用域中，将该函数动态注入到 Scene 类，例如：
    def my_func(self, arg):
    Scene.my_func = my_func
  或使用 setattr(Scene, "my_func", my_func)。
- 用户在继承 Scene 的场景类中，可以通过 self.my_func() 直接调用。

## 自定义类（扩展 Manim 原生类）
- 定义新类，必须继承自 manim 的原生类（如 Mobject、Scene、VGroup 等，最好是继承Scene）。
- 该类的实例本质上仍是 Manim 对象（因为是子类），但相对于直接使用原生类构造函数而言，属于“自定义的扩展对象”。
- 用户在代码中通过 obj = MyClass(...) 来实例化。

## 必须遵守的约束
- 扩展文件只能导入 manim、其原生依赖库（如 numpy）以及 Python 标准库，不得导入其他第三方库。
- 扩展文件为单个 .py 文件（例如 my_extension.py）。用户通过 import my_extension 导入（不带 .py 后缀）。
- 用户的主场景类继承自 Scene，使用 construct 方法作为入口。在该方法内部，用户可以直接使用 self.xxx() 调用你注入的函数，或使用 obj = MyClass(...) 调用你定义的类。
- 除非用户明确要求改变方法，否则以上所有规定不能打破。
